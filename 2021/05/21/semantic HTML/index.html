semantic HTML이란

semantic HTML이란, 말 그대로 코드를 보기만 해도 직관적으로 이해할 수 있게 작성한 HTML을 의미한다. 예를 들어, div 태그와 기능은 동일하지만 이름이 다른 태그가 있는데, header 태그와 main 태그 등이 그 예시이다. 이런 것들은 전부 div와 동일한 기능을 하는 태그이지만 좀 더 직관적으로 작성할 수 있다.

가령 다음의 예시를 보자.



header, main, footer 태그는 div로 대체할 수 있으나, 좀 더 직관적으로 어떤 역할을 하는 지 코드만 봐도 알 수 있게 한다. 이것을 semantic 하다고 한다. 개인이 하는 프로젝트는 문제가 없으나, 협업을 해야하는 경우에는 정해진 표기법(카멜 표기법, 스네이크 표기법 등)에 맞추어 작성하고 주석도 잘 달아주어야 한다. semantic HTML 또한 협업에서 매우 중요한 부분이라고 생각하고, 특히 코드를 작성한 지 한참 지나서 다시 봤을 때에도 도움이 될 것이라고 생각한다.
